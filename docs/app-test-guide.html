FinPay 애플리케이션 테스트 가이드

김동규는 앱을 보완하고, 임지혁과 한경윤은 앱 테스트, IAM 권한 테스트, 보안 정책 검증을 수행합니다.

1. 6단계 진행 원칙

순서작업담당설명
1앱 제작 및 보완김동규 + Codex검증용 앱, 화면 가독성, API, 테스트 가이드 정리
2앱 테스트임지혁, 한경윤브라우저, PowerShell, Thunder Client로 정상/비정상 응답 확인
3IAM 권한 테스트임지혁 중심정상 접근과 비정상 접근, RDS/Secrets/KMS 관련 증빙 확인
4보안 정책 검증한경윤 중심ALB, WAF, Security Group, EventBridge, SNS, 탐지 규칙 확인
5감사 로그 분석 및 시각화임지혁, 한경윤테스트가 끝난 뒤 CloudTrail, CloudWatch, Config, SNS 결과를 분석

감사 로그 분석은 제일 마지막에 진행합니다. 먼저 앱 테스트와 보안 검증으로 로그와 이벤트를 충분히 만든 뒤 분석합니다.

2. 현재 테스트 환경 구성 상태

항목상태설명
브라우저 화면 테스트가능/payments/, /auth/, /ops/ 화면에서 역할별 접근 상태를 확인합니다.
앱 내부 가이드가능/payments/guide, /auth/guide, /ops/guide에서 테스트 순서를 확인합니다.
Thunder Client API 테스트가능GET, POST, PATCH 요청으로 정상/비정상 접근을 검증합니다.
IAM 권한 테스트가능앱의 403 응답과 AWS IAM/Secrets/KMS 조회 결과를 함께 증빙합니다.
보안 정책 검증가능ALB, WAF, Security Group, EventBridge, SNS 구성을 AWS 콘솔 또는 CLI로 확인합니다.
감사 로그 분석마지막 수행테스트로 이벤트를 만든 뒤 CloudTrail, CloudWatch, Config, SNS 결과를 분석합니다.

현재 앱은 실제 Cognito ID/PW 로그인과 JWT 검증을 수행하지 않습니다. 역할 선택은 ?role=Auditor 같은 테스트용 파라미터로 처리합니다.

3. 담당자별 역할 배분

담당자역할세부 작업산출물
김동규 앱 제작 검증용 앱 보완, 화면 가독성 개선, 앱 내부 가이드 제공, API 동작 확인 앱 화면, 앱 API 목록, 테스트 가이드
임지혁 앱 테스트 + IAM 권한 테스트 역할별 정상/비정상 접근 테스트, RDS DB Check, Secrets/KMS/IAM 권한 확인 앱 테스트 결과표, IAM 정상/비정상 접근 캡처, RDS/Secrets/KMS 증빙
한경윤 앱 테스트 + 보안 정책 검증 Ops 앱 테스트, ALB/WAF/SG 정책 검증, EventBridge/SNS 탐지 규칙 확인 보안 정책 검증표, 정책 확인 캡처, 탐지 규칙 증빙

4. 테스트 준비

ALB DNS를 먼저 확인합니다.

$alb = terraform output -raw alb_dns_name

브라우저에서 사용할 기본 주소는 http://<ALB_DNS> 입니다.

9. 임지혁 테스트 항목

앱 테스트

curl "http://$alb/payments/health"
curl "http://$alb/payments/config"
curl "http://$alb/payments/db-check"
curl "http://$alb/payments/api/session?role=Customer"
curl "http://$alb/payments/api/session?role=Auditor"
curl -Method POST "http://$alb/payments/api/payments?role=Customer"
curl -Method POST "http://$alb/payments/api/payments?role=Auditor"

정상 기준: Customer는 결제 요청 생성 가능, Auditor는 Payment에서 차단되어 403 또는 allowed=false가 확인되어야 합니다.

IAM 권한 테스트

항목정상 기준
RDS Public Access비활성화 상태
RDS DB Check앱 서버에서 5432 reachable
Secrets Manager앱 역할 또는 허용된 역할만 접근 가능
KMSRDS/Secrets/Logs 암호화 키 적용 확인
비정상 접근권한 없는 역할은 AccessDenied 또는 403 확인

10. 한경윤 테스트 항목

앱 테스트

curl "http://$alb/ops/health"
curl "http://$alb/ops/config"
curl "http://$alb/ops/api/session?role=Auditor"
curl "http://$alb/ops/api/session?role=Customer"
curl -Method POST "http://$alb/ops/api/audit?role=Auditor&type=manual-audit-test"
curl "http://$alb/ops/api/audit"

정상 기준: Auditor는 Ops에서 허용되고, Customer는 Ops에서 차단되어야 합니다.

12. 감사 로그 분석 및 시각화

이 항목은 앱에서 먼저 수행하지 않습니다. 앱 테스트, IAM 권한 테스트, 보안 정책 검증이 끝난 뒤 마지막에 수행합니다.

대상확인 내용
CloudTrail테스트 중 발생한 API 호출 기록
CloudWatch Logs앱 이벤트, 감사 이벤트, 서비스 로그
EventBridge/SNS탐지 규칙과 알림 이벤트
AWS Config컴플라이언스 규칙 평가 결과
