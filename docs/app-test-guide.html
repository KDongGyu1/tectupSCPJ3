FinPay 6단계 앱 테스트 가이드

기준 역할은 Customer, Merchant, SettlementOperator, Auditor, OperationsAdmin 5개입니다.

1. 접속 주소

항목주소용도
최종 앱https://d1lqumb0x9ifau.cloudfront.net브라우저 기반 앱 테스트
ALB Healthhttp://finpay-dev-alb-1447356258.ap-northeast-2.elb.amazonaws.com/healthALB와 앱 서버 상태 확인
시스템 상태/system/statusRDS, Secrets Manager, CloudWatch 연동 확인

2. 테스트 계정

이메일역할주요 확인 항목
customer@finpay.localCustomer결제 요청 생성, 본인 결제 조회
merchant@finpay.localMerchant가맹점 결제 요청 생성, 관련 결제 조회
settlement@finpay.localSettlementOperator결제 승인 또는 거절
auditor@finpay.localAuditor결제 내역과 감사 이벤트 조회
ops@finpay.localOperationsAdmin운영, 승인, 감사, 시스템 상태 확인

회원가입은 비활성화합니다. 테스트 계정은 관리자 CLI에서 만들고 Cognito 그룹으로 역할을 부여합니다.

3. 임지혁 담당 테스트

앱 기능 및 IAM 권한 테스트

테스트정상 기준
Customer 로그인 후 결제 생성결제 생성 성공, 감사 이벤트에 Success 기록
Auditor로 결제 생성 시도403 또는 권한 없음 화면 표시, 감사 이벤트에 Denied 기록
SettlementOperator로 결제 승인/거절Pending 결제 상태가 Approved 또는 Rejected로 변경
RDS 상태 확인/system/status에서 RDS reachable 표시
Secrets Manager 상태 확인/system/status에서 configured 표시

4. 윤정우 담당 테스트

보안 정책 검증

테스트정상 기준
Cognito 실제 로그인로그인 후 앱 우측 상단에 이메일과 역할 표시
역할별 메뉴 노출 확인권한 없는 메뉴는 보이지 않거나 접근 시 403 표시
Auditor 감사 이벤트 조회감사 이벤트 목록 조회 가능, 결제 생성/승인은 불가
OperationsAdmin 시스템 상태 조회시스템 상태, 보안 상태, 감사 이벤트 조회 가능
CloudWatch 로그 확인로그인, 권한 거부, 결제 이벤트가 구조화 로그로 기록
