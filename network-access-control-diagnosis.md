# 네트워크/접근통제 진단표

기준: Terraform 코드, 저장소 문서, AWS CLI 조회 결과를 함께 사용했다. 코드에서 확인되지 않는 실제 런타임 상태는 추가 확인 명령어로 분리했다.

| 점검 항목 | 현재 구성 | 정상 기준 | 현재 판단 | 위험도 | 개선 필요 여부 | 근거 파일 | 추가 확인 명령어 |
|---|---|---|---|---|---|---|---|
| VPC | 코드: `10.0.0.0/16`, DNS support/hostname 활성화. CLI: `finpay-dev-vpc` 확인. | 프로젝트 전용 VPC가 명확하고 CIDR, 태그, 리전이 일치해야 한다. | 정상 | 낮음 | 아니오 | `modules/network/main.tf`, `variables.tf` | `aws ec2 describe-vpcs --region ap-northeast-2` |
| Public/App/DB Subnet | 코드: Public `10.0.0.0/24`, `10.0.1.0/24`; App `10.0.10.0/24`, `10.0.11.0/24`; DB `10.0.20.0/24`, `10.0.21.0/24`. 모두 2-AZ 구성. | Public, App, DB Subnet이 용도별로 분리되고 2개 이상 AZ에 배치되어야 한다. | 정상 | 낮음 | 아니오 | `modules/network/main.tf`, `modules/network/locals.tf`, `variables.tf` | `aws ec2 describe-subnets --region ap-northeast-2 --filters Name=vpc-id,Values=$VPC_ID` |
| Subnet Public IP 자동 할당 | 코드: Public/App/DB 모두 `map_public_ip_on_launch = false`. CLI: 모든 Subnet에서 `PublicIpOnLaunch=false` 확인. | App/DB Subnet은 Public IP 자동 할당이 꺼져 있어야 한다. | 정상 | 낮음 | 아니오 | `modules/network/main.tf` | `aws ec2 describe-subnets --region ap-northeast-2 --filters Name=vpc-id,Values=$VPC_ID` |
| Route Table 분리 | 코드: Public, App, DB Route Table 분리. Public/App은 각각 association, DB는 local-only Route Table association. | Subnet 계층별 Route Table이 분리되어야 한다. | 정상 | 낮음 | 아니오 | `modules/network/main.tf` | `aws ec2 describe-route-tables --region ap-northeast-2 --filters Name=vpc-id,Values=$VPC_ID` |
| Public Subnet IGW 연결 | 코드: Public RT에 `0.0.0.0/0` -> IGW. CLI: IGW attached 확인. | Public Subnet만 IGW 기본 경로를 가져야 한다. | 정상 | 낮음 | 아니오 | `modules/network/main.tf` | `aws ec2 describe-internet-gateways --region ap-northeast-2 --filters Name=attachment.vpc-id,Values=$VPC_ID` |
| App Subnet NAT Gateway 사용 | 코드: App RT에 AZ별 NAT Gateway 기본 경로. CLI: NAT Gateway 2개 `available` 확인. | Private App Subnet은 외부 outbound가 필요하면 NAT 또는 VPC Endpoint를 사용해야 한다. | 정상 | 낮음 | 아니오 | `modules/network/main.tf` | `aws ec2 describe-nat-gateways --region ap-northeast-2 --filter Name=vpc-id,Values=$VPC_ID` |
| DB Subnet 인터넷 직접 연결 여부 | 코드: DB RT에는 인터넷 기본 경로 없음. CLI: DB RT에 `0.0.0.0/0` 없음 확인. | DB Subnet은 IGW/NAT로 향하는 인터넷 default route가 없어야 한다. | 정상 | 낮음 | 아니오 | `modules/network/main.tf` | `aws ec2 describe-route-tables --region ap-northeast-2 --filters Name=vpc-id,Values=$VPC_ID` |
| ALB 외부 진입 구조 | 코드: ALB `internal = false`, Public Subnet 배치. CLI: `internet-facing`, Public Subnet 확인. | 외부 진입점은 ALB 또는 CloudFront로 통제되어야 하며 App/DB 직접 노출이 없어야 한다. | ALB 외부 진입 구조는 구현됨. 다만 ALB 직접 접근 가능. | 중간 | 예 | `modules/app/main.tf` | `aws elbv2 describe-load-balancers --region ap-northeast-2 --names finpay-dev-alb` |
| ALB Listener | 코드: HTTP 80 Listener는 항상 생성, HTTPS 443 Listener는 `alb_certificate_arn`이 있을 때만 생성. CLI: 현재 HTTP 80만 확인. | HTTPS 443 우선, HTTP 80은 HTTPS redirect 권장. | 개선 필요 | 중간 | 예 | `modules/app/main.tf`, `variables.tf`, `terraform.tfvars.example` | `aws elbv2 describe-listeners --region ap-northeast-2 --load-balancer-arn $ALB_ARN` |
| ALB Target Group | 코드: Target Group protocol `HTTP`, port `8080`, ASG 연결. CLI: auth/payment/ops Target healthy 확인. | ALB는 Private App 서버로만 트래픽을 전달해야 한다. | 동작 정상, 전송보안 개선 필요 | 중간 | 예 | `modules/app/main.tf` | `aws elbv2 describe-target-groups --region ap-northeast-2 --load-balancer-arn $ALB_ARN` |
| ALB SG | 코드: Inbound 80/443 from `allowed_http_cidr_blocks`, 기본값 `0.0.0.0/0`; egress 8080 to VPC CIDR. | 외부 공개는 정책상 허용된 CIDR 또는 CloudFront 경로로 제한해야 한다. | 주의. ALB DNS 직접 접근 가능. | 중간~높음 | 예 | `modules/security_groups/main.tf`, `variables.tf`, `terraform.tfvars.example` | `aws ec2 describe-security-groups --region ap-northeast-2 --filters Name=group-name,Values=finpay-dev-alb-sg` |
| App SG | 코드: Inbound 8080은 ALB SG에서만 허용. Egress 443 to `0.0.0.0/0`, 5432 to VPC CIDR. CLI: App EC2 Public IP 없음 확인. | App은 ALB SG에서만 접근 가능해야 한다. | 접근통제 정상, egress는 개선 후보 | 중간 | 일부 예 | `modules/security_groups/main.tf`, `modules/app/main.tf` | `aws ec2 describe-security-groups --region ap-northeast-2 --filters Name=group-name,Values=finpay-dev-app-sg` |
| DB SG | 코드: Inbound 5432는 App SG에서만 허용. Egress rule 없음. | DB는 App SG에서만 DB 포트 접근을 허용해야 한다. | 정상 | 낮음 | 아니오 | `modules/security_groups/main.tf` | `aws ec2 describe-security-groups --region ap-northeast-2 --filters Name=group-name,Values=finpay-dev-db-sg` |
| VPC Endpoint SG | 코드: Inbound 443은 App SG에서만 허용, egress all to VPC CIDR. | Interface Endpoint는 Private App 계층에서만 접근 가능해야 한다. | 정상, egress 세분화 후보 | 낮음~중간 | 일부 예 | `modules/security_groups/main.tf` | `aws ec2 describe-security-groups --region ap-northeast-2 --filters Name=group-name,Values=finpay-dev-vpce-sg` |
| VPC Endpoint | 코드: S3 Gateway Endpoint, KMS/Secrets Manager/Logs/SSM/SSM Messages/EC2 Messages Interface Endpoint 구성. CLI: Endpoint available 확인. | Private 리소스가 주요 AWS 서비스 접근 시 인터넷 의존도를 줄여야 한다. | 정상, Endpoint Policy 강화 필요 | 중간 | 예 | `modules/network/main.tf`, `modules/vpc_endpoints/main.tf` | `aws ec2 describe-vpc-endpoints --region ap-northeast-2 --filters Name=vpc-id,Values=$VPC_ID` |
| WAF | 코드: Regional WebACL, ALB association, Managed Rules와 Rate Rules 구성. CLI: ALB에 WebACL 연결 확인. | ALB 또는 CloudFront에 WAF가 연결되고 SQLi/공통 공격/Rate 제한이 있어야 한다. | ALB WAF는 정상, CloudFront WAF는 미구현 | 중간 | 예 | `modules/waf/main.tf` | `aws wafv2 get-web-acl-for-resource --region ap-northeast-2 --resource-arn $ALB_ARN` |
| WAF Rule 동작 | 코드: AWS Managed Rule은 `override_action none`, Rate Rule은 `count`. | 탐지 안정화 후 고위험 경로는 Block 전환 기준이 있어야 한다. | 부분 적용 | 중간 | 예 | `modules/waf/main.tf` | `aws wafv2 get-web-acl-for-resource --region ap-northeast-2 --resource-arn $ALB_ARN` |
| CloudFront | 코드: CloudFront Distribution 생성, ALB Origin 사용, Viewer는 `redirect-to-https`, Origin은 `http-only`. CLI: 실제 OriginProtocolPolicy `http-only`, WebACL 없음 확인. | Viewer는 HTTPS 강제, Origin도 HTTPS 권장. CloudFront WAF 적용 검토 필요. | 부분 적용, 심화 개선 과제 | 중간 | 예 | `modules/app/main.tf`, `outputs.tf` | `aws cloudfront get-distribution-config --id E2V5455X4MM78W` |
| Route 53 / ACM | 코드: Route 53 리소스 없음, ALB 인증서는 변수만 존재하고 기본값 빈 값. | 운영 도메인과 ACM 인증서를 사용해 HTTPS 종단을 명확히 해야 한다. | 코드상 미구현, 심화 개선 과제 | 중간 | 예 | `variables.tf`, `terraform.tfvars.example`, `modules/app/main.tf` | `aws acm list-certificates --region ap-northeast-2` |
| 보안그룹 변경 탐지 | 코드: EventBridge Rule이 SG 변경 API를 SNS로 전달. CLI: Rule enabled, SNS target, email subscription 확인. | SG 변경 API가 CloudTrail 기반 EventBridge Rule로 탐지되고 수신자가 있어야 한다. | 정상 | 낮음 | 수신 테스트 필요 | `modules/automation/main.tf`, `modules/logging/main.tf` | `aws events describe-rule --region ap-northeast-2 --name finpay-dev-security-group-changes` |

## 요약 판단

- 네트워크 계층 분리, App/DB Private 유지, DB 인터넷 직접 차단은 정상으로 판단된다.
- 가장 큰 개선점은 ALB 직접 접근 가능성과 CloudFront -> ALB 구간 `http-only` 설정이다.
- WAF는 ALB에 연결되어 있으나 Rate Rule이 Count 모드이고 CloudFront WebACL은 없다.
- ACM, Route 53, CloudFront WAF, ALB HTTPS 전환은 심화 개선 과제로 분류한다.
